## Problem

You could generate question papers successfully, but when you tried to take a test:
- ❌ The topic you just generated papers for didn't appear in the topic dropdown
- ❌ Only hardcoded placeholder topics were showing ("Algebra", "Geometry", etc.)
- ❌ Students couldn't access the papers that were actually generated

---

## Root Cause

The `populate_topics()` method in `student_dashboard.py` was using **hardcoded placeholder topics** instead of discovering actual topics from saved papers:

```python
# OLD CODE ❌
def populate_topics(self):
    placeholder_topics = [
        "Algebra", "Geometry", "Trigonometry", "Calculus",
        "Photosynthesis", "Cellular Biology", "Genetics", "Evolution"
    ]
    # For demonstration, just use all topics
    self.topic_combo.addItems(placeholder_topics[:4])
```

This meant:
- Real topics generated by admin were ignored
- Only 4 hardcoded topics ever showed up
- No dynamic topic discovery

---

## Solution

### Part 1: Add Topic Discovery Method

**File**: `app/services/question_service.py`

Added new method: `get_available_topics(grade, subject)`

```python
def get_available_topics(self, grade: str, subject: str) -> List[str]:
    """Discover available topics by scanning saved papers"""
    topics = set()
    
    try:
        if not os.path.exists(BASE_DATA_PATH):
            return []
        
        # Scan all JSON files in the data directory
        for filename in os.listdir(BASE_DATA_PATH):
            if filename.startswith('paper_') and filename.endswith('.json'):
                # Parse filename: paper_Grade_Subject_Topic_p1.json
                parts = filename.replace('paper_', '').replace('.json', '').split('_')
                
                if len(parts) >= 3:
                    file_grade = parts[0]
                    file_subject = parts[1]
                    # Topic is everything except the last part (which is paper number pX)
                    file_topic = '_'.join(parts[2:-1]).replace('_', ' ')
                    
                    # Match grade and subject
                    if file_grade == grade and file_subject == subject:
                        if file_topic and self.get_available_papers(grade, subject, file_topic):
                            topics.add(file_topic)
        
        return sorted(list(topics))
    except Exception as e:
        print(f"Error discovering topics: {e}")
        return []
```

**What it does:**
1. Scans the `app/data/` directory for saved paper files
2. Extracts grade, subject, and topic from filenames
3. Filters by matching grade and subject
4. Returns sorted list of unique topics
5. Only includes topics that have at least one paper

### Part 2: Update Student Dashboard

**File**: `app/ui/student_dashboard.py`

Replaced `populate_topics()` method:

```python
# NEW CODE ✅
def populate_topics(self):
    """Populate available topics for selected grade and subject"""
    self.topic_combo.clear()
    
    grade = self.grade_combo.currentText()
    subject = self.subject_combo.currentText()
    
    if not grade or not subject:
        return
    
    # Get available topics from saved papers
    available_topics = self.question_service.get_available_topics(grade, subject)
    
    if available_topics:
        self.topic_combo.addItems(available_topics)
    else:
        self.topic_combo.addItem("No topics available")
```

**What changed:**
- ✅ Removed hardcoded placeholder topics
- ✅ Calls new `get_available_topics()` method
- ✅ Dynamically discovers real topics
- ✅ Shows helpful message when no topics exist
- ✅ Repopulates when grade/subject changes

  → Correct behavior
